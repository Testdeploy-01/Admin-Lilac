# 📌 คู่มือและรายละเอียดฟังก์ชัน Lilac Admin Dashboard (ฉบับเจาะลึก)

เอกสารนี้รวบรวมรายละเอียดของหน้าจอ (Pages) ส่วนประกอบ (Components) และฟังก์ชันการทำงาน (Functions) ทั้งหมดที่มีในระบบ Lilac Admin Console เพื่อใช้เป็นแหล่งอ้างอิงในการใช้งานและการพัฒนาต่อเนื่อง

---

## 🧭 แถบเมนูด้านข้าง (Sidebar) และ แถบเครื่องมือด้านบน (Topbar)
### 1. แถบเมนูด้านข้าง (Sidebar Menu)
ทำหน้าที่เป็นตัวนำทางหลักของระบบ แบ่งหมวดหมู่การเข้าถึงดังนี้:
*   **หลัก (Main):**
    *   **ภาพรวม (Overview):** หน้า Dashboard แรกเพื่อดูสรุปสถิติทั้งหมด
*   **โมดูลแอป (App Modules):**
    *   **จัดการ AI (AI Monitor):** ตรวจสอบการใช้ทรัพยากร AI และต้นทุน
    *   **การเรียน (Study):** ติดตามสถิติพฤติกรรมการเรียนของนักศึกษาผ่านแอป
    *   **การเงิน (Finance):** ดูข้อมูลพฤติกรรมการจัดการการเงินของผู้ใช้
    *   **ไลฟ์สไตล์ (Lifestyle):** สรุปอารมณ์และกิจวัตรประจำวันของผู้ใช้แอป
*   **ผู้ใช้งาน (User Management):**
    *   **ผู้ใช้ (Users):** จัดการบัญชีผู้ใช้ (มี Badge บอกจำนวนผู้ใช้รอดำเนินการ)
    *   **สมาชิก (Subscriptions):** ดูยอดขาย แพ็กเกจ PLUS+ และใบแจ้งหนี้
*   **ระบบ (System):**
    *   **แจ้งเตือน (Notifications):** ศูนย์รวมและระบบบรอดแคสต์ข้อความ
    *   **ตั้งค่า (Settings):** ปรับจูนระบบและคีย์เวิร์ด API
    *   **บันทึกระบบ (Logs):** ติดตาม Log แบบ Real-time
    *   **ความคิดเห็น (Feedback):** ดูคะแนนและบั๊กที่ผู้ใช้รายงาน

### 2. แถบเครื่องมือด้านบน (Topbar)
*   **ชื่อหน้าปัจจุบัน:** จะเปลี่ยนไปตามเมนูที่คลิก
*   **ค้นหาด่วน (Quick Search):** กด `/` บนคีย์บอร์ดเพื่อค้นหาผู้ใช้ หรือข้อมูลในระบบได้อย่างรวดเร็ว
*   **Actions:**
    *   `สลับโหมด (Theme Toggle)` สลับระหว่าง Light/Dark Mode 
    *   `ดาวน์โหลดข้อมูล` ปุ่มส่งออกข้อมูลภาพรวมของหน้านั้น ๆ 
    *   `หน้าแจ้งเตือนย่อย` ไอคอนกระดิ่งพาไปหน้า Notifications
    *   `รีเฟรช (Refresh)` รีโหลดข้อมูลในหน้าปัจจุบัน

## 📊 รายละเอียดฟังก์ชันแต่ละหน้าต่าง (Pages)

### 1. 👁️ ภาพรวม (Overview)
**จุดประสงค์:** หน้าแรกที่ผู้ดูแลระบบจะเห็น เพื่อสรุปสถิติแบบด่วน (Bird-eye view)
*   **KPI Cards (สถิติหลัก):**
    *   **ผู้ใช้ทั้งหมด:** แสดงยอดรวมและสถิติเทียบรายสัปดาห์
    *   **ผู้ใช้ที่ใช้งานวันนี้ (DAU):** ตัวเลข Active session ทียบกับเมื่อวาน
    *   **แจ้งเตือนที่เปิดใช้งาน:** จำนวนการแจ้งเตือนที่ทำงานอยู่ในระบบ
    *   **ใช้ทางลัดวันนี้ (Shortcut Usage):** การสั่งงาน AI ผ่าน Floating Action Button
*   **กราฟและสถิติ:**
    *   **กราฟการเติบโตผู้ใช้:** กราฟเส้นแสดงสถิติย้อนหลัง 30 วัน
    *   **สัดส่วนการใช้งานตามหมวด:** กราฟวงกลมแยกเปอร์เซ็นต์ (เรียน, เงิน, ไลฟ์สไตล์) พร้อม Badge ไฮไลต์หมวดที่มาแรงที่สุดสัปดาห์นี้
*   **ฟีดข้อมูลแบบสด:**
    *   **กิจกรรมล่าสุด (Activity Feed):** อัปเดตพฤติกรรมผู้ใช้ เช่น ใครเพิ่งสมัครแพ็กเกจใหม่ ใครกำลังเจอปัญหา (คลิกดูทั้งหมดจะพาไปหน้า Users)
    *   **ฟีเจอร์ยอดนิยม (Popular Features):** ลิสต์จัดอันดับฟังก์ชันที่ถูกเรียกใช้ข้ามระบบเยอะที่สุด

### 2. 🧠 จัดการ AI (AI Usage Monitor)
**จุดประสงค์:** ควบคุมต้นทุนโทเคนและประสิทธิภาพของ AI โมเดล
*   **ระบบแจ้งเตือนฉุกเฉิน (Alert Banner):** จะปรากฏขึ้นเมื่อมีผู้ใช้ดึง API สูงเกินลิมิตใน 1 ชั่วโมง
*   **ฟิลเตอร์ดูสถิติ:** เลือกดูแบบ วันนี้, 7 วัน, เดือนนี้, ปีนี้
*   **KPI Cards (สถิติ AI):**
    *   **โทเคนทั้งหมด:** แจกแจงยอดโทเคนเข้า/ออก
    *   **ต้นทุน AI:** รวบยอดค่า API เป็นเงินบาท (แอดมินดูเรท USD ได้)
    *   **ความแม่นยำจัดหมวด:** เปอร์เซ็นต์ที่ระบบตีความคำสั่ง (Intent) สำเร็จ
    *   **กำไรขั้นต้น (Margin):** สัดส่วนต้นทุน AI เทียบกับรายได้
*   **ส่วนวิเคราะห์เชิงลึก:**
    *   **กราฟโทเคน&ต้นทุน:** แท่งกราฟเทียบระหว่าง Tokens และเงินบาท
    *   **สัดส่วนแยกตามหมวด:** วงกลมชี้เป้าว่าหมวดไหนกิน Token มากสุด
    *   **การใช้โทเคนแยกตามโมเดล:** แสดงหลอด Progress bar แบบแยก Model (Claude-Haiku, Sonnet, Opus) พร้อมต้นทุนจริงที่เสียไปแต่ละตัว
*   **ตารางสถิติรายบุคคล (User Details Table):** ลิสต์ว่าผู้ใช้แต่ละคนใช้ Token ขาเข้า-ออกเท่าไร และคำนวณ 'ส่วนต่างกำไร' (Margin) เทียบกับค่าสมาชิกที่จ่ายให้เพื่อเฝ้าระวังสภาวะขาดทุน

### 3. 📚 การเรียน (Study Module)
**จุดประสงค์:** รวบรวมสถิติพฤติกรรมทางการเรียนของกลุ่มเป้าหมาย (นักศึกษา)
*   **KPI Cards:**
    *   สัดส่วนผู้ใช้สร้าง "ตารางเรียน"
    *   จำนวนอีเวนต์สอบ/โปรเจกต์รายสัปดาห์
    *   ความสม่ำเสมอในการเปิดแอปตรงเวลาเรียน
    *   วิชายอดนิยมในระบบ
*   **กราฟวิเคราะห์:**
    *   ยอดรวมงาน/สอบในปฏิทินผู้ใช้ทั้งหมด
    *   สัดส่วนอีเวนต์ Upcoming
*   **ระบบจับกระแส (Trending Events Feed):** 
    *   มีตัวกรองตาม 'มหาวิทยาลัย' และ 'ชั้นปี'
    *   โชว์ลิสต์งานด่วน/การสอบที่จะมาถึง (เช่น Midterm, ส่ง Project) พร้อม Insight ว่า "ผู้ใช้อินบ็อกซ์หา AI ถามเรื่องวิชานี้เยอะผิดปกติ (+22%)" เพื่อใช้โปรโมทหรือส่งกำลังใจ

### 4. 💰 การเงิน (Finance Module)
**จุดประสงค์:** ภาพรวมแพลตฟอร์มในการให้ AI ช่วยจัดการการเงิน
*   **KPI Cards:**
    *   สัดส่วนผู้ใช้ที่คุมงบประมาณอยู่ (On-track)
    *   สัดส่วนผู้ใช้ที่ใช้เงินทะลุงบ
    *   สัดส่วนการใช้ AI ทำบัญชี (แยก แชท vs พิมพ์เสียง)
    *   สัดส่วนการพิมพ์ลงบัญชีเอง (Manual)
*   **ตารางสรุปยอดหมวดหมู่:** แบ่งดู รายรับ/รายจ่าย วัดอันดับหมวดหมู่เป้าหมาย (เช่น อาหาร, ทางรถ, ชอปปิง) พร้อมแท่ง Progress Bar สัดส่วนในระบบ
*   **กราฟวิเคราะห์พฤติกรรม:**
    *   ความถี่ในการบันทึกบัญชีต่อวัน (Bar chart)
    *   เทียบวิธีลงบัญชีกรอกเอง vs ให้ AI กรอก (Donut chart)

### 5. 🌿 ไลฟ์สไตล์ (Lifestyle Module)
**จุดประสงค์:** ดูแลสุขภาพจิต พฤติกรรม และความออร์แกนิกของผู้ใช้
*   **KPI Cards:**
    *   ผู้ใช้ที่มีตาราง Routine
    *   จำนวนไดอารี่/โน้ตที่มีการแนบภาพ
    *   ดัชนีภาพรวมอารมณ์ (Mood Index)
    *   ผู้ใช้ในโหมดสุขภาพ/พักผ่อนน้อย (ระวังความเครียด)
*   **ระบบสถิติคำแนะนำของ AI (AI Recommendations):** มีปุ่มสลับดูลิสต์:
    *   **เมนูอาหาร:** เมนูที่ AI เดาจากอารมณ์แล้วเสนอให้ผู้ใช้กิน พร้อมดู Click-through rate (CTR%) ว่าผู้ใช้เอาเมนูไปทำตามจริงกี่เปอร์เซ็นต์
    *   **ภาพยนตร์ซีรีส์:** หนังเรื่องไหนที่ AI แนะนำแล้วถูกกดดูบ่อยสุด
*   **กราฟอารมณ์/พฤติกรรม:**
    *   สรุปอารมณ์ผู้ใช้แบบองค์รวม
    *   เวลาการตั้งปลุกอัจฉริยะยอดฮิต (เช่น ตื่นก่อนเรียน 1 ชม., เตือนออกกำลังกาย 17:00)

### 6. 👤 ผู้ใช้งาน (User Management)
**จุดประสงค์:** ศูนย์ควบคุมลูกค้า / การจัดการ Account ต่าง ๆ
*   **ระบบกรองผู้ใช้ขั้นสูง:** 
    *   Tabs: ทั้งหมด, PLUS+, FREE, ถูกระงับ (Suspended), ใหม่ 7 วันล่าสุด
    *   Sorting: เรียงตามวันที่สมัคร, เรียงตามการเผาผลาญ Token
    *   Pagination ควบคุมจำนวนคนในตาราง
*   **ตารางฐานข้อมูลผู้ใช้:** แสดงชื่อ, แพลน, วันสมัคร, หมวดที่อินที่สุด, Alert ของระบบ, จำนวน Total Tokens ที่เผาในเดือน, ต้นทุน AI, สถานะบัญชี
*   **Sidebar เด้งด้านข้าง (User Details Panel):**
    *   เมื่อคลิกชื่อใครในตาราง จะมีจอไถลออกมาจากด้านขวา
    *   สามารถกด `แก้ไขแพลน (Edit Plan)` หรือกด `ระงับบัญชี (Suspend)` จากหน้านี้ได้ทันที

### 7. 💳 สมาชิก (Subscriptions / แพ็กเกจ)
**จุดประสงค์:** ด้านธุรกิจและการทำตั้งค่าตัวเลข Subscription
*   **แท็บการทำงานหลัก:** ภาพรวม, แพ็กเกจ, ใบแจ้งหนี้
    *   **ภาพรวม (Overview):** ดูสถานะทางการเงิน คาดการณ์รายได้ MRR, ยอดผู้ใช้ PLUS+, อัตราการ Conversion ของระบบ
    *   **แพ็กเกจ (Plans):** ระบบตั้งค่า Package ขายของ (รายเดือน, รายเทอม, รายปี) 
        *   ตั้งราคา, ระยะเวลาที่หมดอายุ, บังคับเปิด/ปิดแพ็กเกจ
        *   ตั้งค่าส่วนกลาง เช่น Free Trial กี่วัน, โทเคนผู้ใช้เริ่มต้น
        *   `ตั้งค่าสิทธิพิเศษ PLUS+`: พิมพ์คำอธิบายและบังคับโชว์ติ๊กถูกหน้าบ้านแอป เพื่อกระตุ้นการขาย
    *   **ใบแจ้งหนี้ (Invoices):** ตรวจสอบและดาวน์โหลด Statement ใบเสร็จลูกค้าแต่ละคน

### 8. 🔔 แจ้งเตือน (Notifications / Broadcast)
**จุดประสงค์:** ศูนย์กระจายข่าวสารสู่โทรศัพท์ผู้ใช้งานและเช็กประวัติความเคลื่อนไหว
*   **หมวดหมู่จดหมาย:** ดูแบบ รวม, สำคัญ, ระบบ, และการเงิน
*   **ฟอร์มสร้างการแจ้งเตือน (Broadcaster Tool):**
    *   ช่องสำหรับ: หัวข้อ, รายละเอียด
    *   ออปชันเลือกหมวด (ใส่ไอคอนหมวดหมู่)
    *   ออปชันฐานผู้ชม: ส่งทุกคน, ส่งแค่ Plus, ส่งแค่สายฟรี หรือยิงแอดกลุ่มเฉพาะเจาะจง
    *   กด "ส่งทันที" หรือ "เซฟแบบร่างไว้" (Draft)
*   **Log ถาดออก:** ตารางเช็กบิลล์ว่าข้อความชุดนี้เจาะกลุ่มเป้าหมายไหน ยิงไปตอนโมงยามไหน

### 9. ⚙️ ตั้งค่า (System Settings)
**จุดประสงค์:** ปลีกย่อยการปรับจูนระบบของแอดมินและการวางโครงสร้างแอป
*   **General Settings:** เปลี่ยนชื่อโครงการ, URL, อีเมลฝ่ายซัพพอร์ต, ปุ่มเช็กการเตือนความผิดพลาดของระบบเข้าอีเมลแอดมิน
*   **API / AI Settings:**
    *   ช่องสำหรับใส่ `Anthropic API Key` (ซ่อนอักขระเป็นดอกจัน)
    *   เลือก Model เริ่มต้น (Default fallback model)
*   **กฎจริยธรรมระบบ (Safety):**
    *   เปิดโหมด NSFW Check ปัดตกคำหยาบหรือการละเมิด
    *   เปิดโหมดยกระดับ Auto-Suspend ระงับชั่วคราวคนใช้ Token หนัก 150% ของปกติ
*   *มีแผงเมนูด้านล่างปรากฏขึ้นทุกครั้งที่มีการกดเปลี่ยนค่า เพื่อให้ติ๊ก บันทึก/ละทิ้ง*

### 10. 📝 บันทึกระบบ (System Logs)
**จุดประสงค์:** จอมอนิเตอร์ของนักพัฒนา (Real-time Tracker)
*   สเตตัสกระพริบเขียวแสดงสถานะ Live Feed
*   **เครื่องมือค้นหาใน Log:** ค้นหาตามรหัส Erorr Code หรือ User ID
*   มีปุ่ม `Pause` เพื่อหยุดจอไหล เวลาต้องการอ่านสคริปต์ยาว ๆ
*   มีปุ่มโหลด .txt หรือ JSON สำหรับส่งต่อหน้างาน

### 11. 💬 ความคิดเห็น (Feedback)
**จุดประสงค์:** คัดกรองเสียงสะท้อนจากผู้ใช้เพื่อพัฒนาต่อ
*   **KPI Cards:** รวม Feedback ทั้งหมด, เรตติ้งเฉลี่ยดาว 5.0, รายงานบั๊กในระบบ (Bug Reports) ที่รอแก้
*   **แท็บคัดกรองตาราง:** คำชม, ข้อเสนอแนะ, ปัญหา(บั๊ก)
*   **ตารางแสดงคิว:** แสดงหน้าตาของผู้พิมพ์ข้อความ, ดาวชี้วัด, ประเภทและข้อความฉบับเต็ม รวมถึงสถานะ (รอดำเนินการ, แก้แล้ว)
